Lecture 4 - JavaScript Language Fundamentals

Lecture 4 - JavaScript Language Fundamentals. Duration: 2 hours. This lecture continues from Lecture 3. It starts with a short recap of HTML and CSS topics from earlier weeks (box model, selectors and specificity, display, positioning), then introduces JavaScript as a high-level, dynamic, untyped, interpreted language suited to both object-oriented and functional styles. The focus is on the language itself—types, control flow, objects, arrays, functions, classes, modules, and regular expressions—before dedicated DOM and browser API work in labs. Connected to unit C4 and Lab 8, Lab 9 (see Schedule).

Learning outcomes

By the end of this lecture you will be able to:

  • Describe JavaScript lexical rules: Unicode, case sensitivity, comments, literals, identifiers, reserved words, and consistent use of semicolons
  • Distinguish primitive types from objects; explain mutability, comparison by value vs reference, and type coercion pitfalls
  • Use let, const, and var appropriately; explain block scope, hoisting, and lexical scoping
  • Evaluate expressions with arithmetic, comparison (=== vs ==), logical, ternary, and typeof operators; reason about precedence
  • Write conditional and loop statements; use break, continue, return, and try/catch/finally
  • Create and traverse objects; understand prototypes, property descriptors, accessors, and JSON serialization limits
  • Use arrays effectively: indexing, length, and core methods (push/pop, splice, map/filter/reduce, sorting pitfalls)
  • Declare and invoke functions; explain this in different call patterns; describe closures and higher-order functions
  • Define classes with extends and super; import and export ES modules
  • Build and test simple regular expressions for validation

Topics

  • Warm-up (HTML/CSS): box model (margin, padding, border); selectors and specificity; display (block, inline, flex); positioning (static, relative, absolute, fixed)
  • Language overview: why JS is high-level, dynamic, untyped, interpreted
  • Lexical structure: character set, comments, literals, identifiers, reserved words
  • Types, values, and variables: primitives vs objects; built-in object types; coercion; global object (window / globalThis)
  • Numbers, strings, booleans: literals, falsy/truthy values, string methods (immutability)
  • Expressions and operators: property access, creation patterns, invocation and this; arithmetic and assignment; comparison and logical operators; typeof, in, instanceof, delete; precedence; avoiding eval on untrusted input
  • Statements: declarations; conditionals; loops; jump statements; exceptions
  • Objects: literals, constructors, Object.create; prototype chain; properties, enumeration, accessors, descriptors, Object.freeze
  • JSON: stringify / parse; what is not serialized
  • Arrays: creation, sparse arrays, mutating methods, searching, sorting (numeric compare functions), higher-order iteration
  • Functions: declarations, expressions, arrows; parameters and defaults; invocation modes; call / apply; scope and closures
  • Classes: syntax, inheritance, prototype chain; modules: export / import
  • Regular expressions: literals and flags; common metacharacters; test, string methods, capture groups; practical validation
  • Quiz and recap: note—an announced quiz will take place in Lecture 5 (see course platform for details).

Bridge from CSS

Before new material, we consolidate Lecture 2 and Lecture 3: how the box model affects size and spacing; how specificity resolves conflicting rules; how display and positioning take elements in or out of normal flow. This sets up using JavaScript later to read computed styles and change classes on the DOM.

JavaScript at the core

Variables hold values; primitives (number, string, boolean, null, undefined, symbols, BigInt) are immutable and compared by value, while objects are mutable and compared by reference. Automatic type coercion means expressions like 1 + "1" and "5" - 1 need careful reading—prefer explicit conversions when clarity matters. Use === and !== by default to avoid coercion surprises in equality tests.

Lexical scoping ties visibility to where code is written; closures let inner functions retain access to outer bindings, which enables patterns like private state in factories. Arrow functions do not have their own this; they inherit it from the enclosing scope, which differs from traditional functions used as methods or constructors.

Lexical structure

  • Character set: JavaScript uses Unicode.
  • Case sensitivity: myVar ≠ myvar ≠ MYVAR.
  • Comments: single-line (//) and multi-line (/* ... */).
  • Literals: values written directly in code (e.g. true, 5, "five", null).
  • Identifiers: start with a letter, _, or $ (then letters/digits too).
  • Reserved words: cannot be used as identifiers (e.g. break, if, while).
  • Semicolons: we use them consistently to avoid subtle ASI edge cases.
// Single-line comment
/* Multi-line
   comment */

const ok = true;
const n = 5;
const word = "five";

Types, values, variables, and coercion

Two big categories:

  • Primitive values (immutable): numbers, strings, booleans, null, undefined (and also symbol, BigInt).
  • Objects (mutable, reference type): collections of properties, arrays, functions, dates, regexes, errors.
// Compared by VALUE (primitives)
let a = "hello";
let b = "hello";
console.log(a === b); // true

// Compared by REFERENCE (objects)
let o1 = { x: 1 };
let o2 = { x: 1 };
console.log(o1 === o2); // false
let o3 = o1;
console.log(o3 === o1); // true

Type coercion is automatic conversion. It is convenient but can surprise you.

console.log(1 + "1");      // "11" (number -> string)
console.log("5" - 1);      // 4 (string -> number)
console.log(null >= 0);    // true (null -> 0 here)
console.log(null == 0);    // false (different rule here)

Scope and declarations

  • let: block scoped.
  • const: block scoped, cannot be reassigned.
  • var: function scoped, hoisted; legacy.
// Hoisting with var (declaration hoisted, initialization not)
console.log(x); // undefined (not an error)
var x = 5;

// Block scope with let/const
if (true) {
  let y = 10;
  const z = 20;
}
// console.log(y); // ReferenceError

Lexical scoping (defined vs called)

let x = "global";
function outer() {
  let x = "outer";
  function inner() {
    let x = "inner";
    console.log(x); // "inner"
  }
  inner();
  console.log(x); // "outer"
}
outer();
console.log(x); // "global"

Numbers, strings, booleans

Numbers

Numbers are 64-bit floating point (IEEE 754). JS does not throw for overflow or division by zero.

console.log(1 / 0);      // Infinity
console.log(-1 / 0);     // -Infinity
console.log(0 / 0);      // NaN

let n = 42;
n += 5; // shortcut
n++;    // post-increment
--n;    // pre-decrement

console.log(Math.pow(2, 8));  // 256
console.log(Math.sqrt(16));   // 4
console.log(Math.round(4.6)); // 5
console.log(Math.random());   // 0..1

Strings

Strings are immutable; methods return new strings.

let s = "Hello";
console.log(s[0]);         // "H"
console.log(s.length);     // 5
console.log(s.toUpperCase()); // "HELLO"
console.log(s);            // still "Hello"

let name = "Alice";
console.log(`Hi ${name}!`); // template literal

Booleans and truthy/falsy

Falsy values: false, 0, -0, "", null, undefined, NaN. Everything else is truthy (including [] and {}).

console.log(Boolean(""));   // false
console.log(Boolean([]));   // true
console.log(Boolean({}));   // true

Globals (browser)

In the browser, the global object is window (modern code can use globalThis). Some globals:

  • Properties: undefined, Infinity, NaN
  • Functions: isNaN, parseInt, parseFloat, eval
  • Objects: Math, JSON
// In the browser:
console.log(parseInt === window.parseInt); // true

// Top-level var becomes a property of window (avoid)
var legacy = 5;
console.log(window.legacy); // 5

Expressions and operators

Expressions evaluate to values; operator precedence matters (use parentheses when in doubt).

// Equality: prefer === / !==
console.log(0 == false);   // true (coercion)
console.log(0 === false);  // false

// Logical operators return values (short-circuit)
console.log(null && "x");  // null
console.log("" || "guest"); // "guest"

// Ternary
let age = 20;
console.log(age >= 18 ? "adult" : "minor");

// typeof
console.log(typeof 42);        // "number"
console.log(typeof "hi");      // "string"
console.log(typeof null);      // "object" (historic bug)
console.log(typeof (() => {})); // "function"

Special operators:

// in / instanceof
console.log("x" in { x: 1, y: 2 }); // true
console.log(new Date() instanceof Date); // true

// delete
let obj = { x: 1, y: 2 };
delete obj.x;
console.log(obj); // { y: 2 }

eval: do not use with untrusted input (security + performance risk).

Operator precedence (high → low)

  1. Property access & invocation: obj.x, f()
  2. Unary operators: !, ++, --, typeof
  3. Arithmetic: * / % then + -
  4. Comparison: > >= < <=
  5. Equality: == === != !==
  6. Logical AND: &&
  7. Logical OR: ||
  8. Ternary: ?:
  9. Assignment: = += etc.

Statements: conditionals, loops, jumps, exceptions

// if/else
if (age >= 18) {
  console.log("adult");
} else {
  console.log("minor");
}

// switch (always use break)
let day = "Mon";
switch (day) {
  case "Mon":
    console.log("Start of week");
    break;
  default:
    console.log("Other day");
}

// while / do..while / for
let x = 0;
while (x < 3) x++;

do { x++; } while (x < 5);

for (let i = 0; i < 3; i++) {
  console.log(i);
}

// for..in (enumerates enumerable properties; includes inherited ones)
let p = { name: "Alice", age: 25 };
for (let key in p) {
  if (p.hasOwnProperty(key)) {
    console.log(key + ": " + p[key]);
  }
}

// break / continue / return
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  if (i === 2) continue;
}

// try/catch/finally
function divide(a, b) {
  if (b === 0) throw new Error("Division by zero!");
  return a / b;
}
try {
  divide(10, 0);
} catch (e) {
  console.log(e.message);
} finally {
  console.log("Done");
}

// Labels (rare, but useful to break out of nested loops)
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (j === 1) break outer;
  }
}

Objects, prototypes, descriptors, JSON

Object literals and methods

let person = {
  name: "Alice",
  age: 25,
  greet: function () {
    return \"Hi, I'm \" + this.name;
  }
};
console.log(person.name);
console.log(person[\"age\"]);
console.log(person.greet());

Creating objects: literal, constructor, Object.create

let p1 = { x: 1, y: 2 };
let d = new Date();
let proto = { breathes: true };
let dog = Object.create(proto);
dog.name = \"Rex\";
console.log(dog.breathes); // inherited

Accessors and property descriptors

let circle = {
  radius: 5,
  get area() { return Math.PI * this.radius ** 2; },
  set area(a) { this.radius = Math.sqrt(a / Math.PI); }
};
console.log(circle.area);
circle.area = 100;
console.log(circle.radius);

Object.defineProperty(person, \"id\", {
  value: 1,
  writable: false,
  enumerable: false,
  configurable: false
});

// Inspect a descriptor
console.log(Object.getOwnPropertyDescriptor(person, "name"));

If you want to prevent changes, you can freeze an object (makes its properties non-writable and non-configurable).

Object.freeze(person);
// person.age = 99; // silently fails or throws in strict mode

JSON

JSON serializes own enumerable data properties. Functions and undefined are not serialized.

let data = { name: \"Alice\", age: 25 };
let json = JSON.stringify(data);
let obj = JSON.parse(json);
console.log(obj.name);

Arrays

// Creation
let empty = [];
let numbers = [1, 2, 3];
let mixed = [1, \"hello\", true, null, { x: 1 }];
let nested = [[1, 2], [3, 4]];

// Access + length
console.log(numbers[0]);   // 1
console.log(numbers.length); // 3

// Length behavior
let t = [1, 2, 3];
t.length = 5; // [1,2,3, empty ×2]
t.length = 2; // [1,2] (truncates)
t.length = 0; // [] (clears)

// push/pop/unshift/shift
numbers.push(4); // [1,2,3,4]
numbers.pop();   // [1,2,3]

// splice: insert/remove/replace
let a = [1, 2, 3, 4, 5];
a.splice(1, 2);        // remove 2 items at index 1 -> [1,4,5]
a.splice(1, 0, 9, 8);  // insert -> [1,9,8,4,5]

// searching
console.log([1,2,3,2].indexOf(2));     // 1
console.log([1,2,3,2].lastIndexOf(2)); // 3
console.log([1,2,3].includes(3));      // true

Note: delete a[i] leaves a hole (sparse array). Use splice to remove elements and shift indexes.

// sorting: default is string sort, so pass compare for numbers
let nums = [3, 1, 20, 2];
nums.sort();                 // [1, 2, 20, 3] (as strings)
nums.sort((x, y) => x - y);  // [1, 2, 3, 20]

// higher-order methods (non-mutating)
let list = [3, 1, 4, 1, 5, 9];
console.log(list.filter(x => x > 3));
console.log(list.map(x => x * 2));
console.log(list.reduce((sum, x) => sum + x, 0));

Functions, this, closures, higher-order functions

Declaring functions

// declaration (hoisted)
function add(a, b) { return a + b; }

// expression (not hoisted)
const add2 = function (a, b) { return a + b; };

// arrow
const add3 = (a, b) => a + b;

Invoking and this

// 1) as a function
function whoAmI() { return this; }
console.log(whoAmI()); // window (or undefined in strict mode)

// 2) as a method
let user = {
  name: \"Alice\",
  greet: function () { return \"Hi, \" + this.name; }
};
console.log(user.greet()); // this => user

// 3) as a constructor
function Person(name) { this.name = name; }
let p = new Person(\"Bob\");
console.log(p.name);

// 4) call/apply
function greet(greeting) { return greeting + \", \" + this.name; }
console.log(greet.call(user, \"Hello\"));
console.log(greet.apply(user, [\"Hello\"]))

Arrow functions do not have their own this; they capture it from the surrounding scope.

Closures

function makeCounter() {
  let count = 0;
  return function () {
    count++;
    return count;
  };
}
const counter = makeCounter();
console.log(counter()); // 1
console.log(counter()); // 2

Higher-order functions

function multiplier(factor) {
  return (x) => x * factor;
}
const double = multiplier(2);
console.log(double(5)); // 10

Classes and modules

Classes and inheritance

class Animal {
  constructor(name) { this.name = name; }
  speak() { return this.name + \" makes a sound.\"; }
}
class Dog extends Animal {
  speak() { return this.name + \" barks.\"; }
  fetch() { return super.speak() + \" Fetching!\"; }
}
let dog = new Dog(\"Rex\");
console.log(dog.speak());
console.log(dog.fetch());

Class declarations are not hoisted; always use new for class constructors.

ES modules

// math.js
export const PI = 3.14159;
export function add(a, b) { return a + b; }
export default class Calculator {
  multiply(a, b) { return a * b; }
}

// main.js
import Calculator, { PI, add } from \"./math.js\";
import * as MathStuff from \"./math.js\";

Regular expressions

RegExp patterns match text. Common flags: i (case-insensitive), g (global/all occurrences).

let pattern = /hello/i;
console.log(pattern.test(\"Hello World\")); // true

let str = \"Hello World Hello\";
console.log(str.match(/hello/gi)); // [\"Hello\", \"Hello\"]
console.log(str.replace(/hello/gi, \"Hi\")); // \"Hi World Hi\"
console.log(str.split(/\\s+/)); // split on whitespace

// capture groups
let date = \"2026-04-14\";
let m = date.match(/(\\d{4})-(\\d{2})-(\\d{2})/);
console.log(m[1]); // year
console.log(m[2]); // month
console.log(m[3]); // day

// simple validation example
function validateEmail(email) {
  let p = /^[\\w.-]+@[\\w.-]+\\.\\w{2,}$/;
  return p.test(email);
}

Tip: test patterns quickly with regex101.
