Flexbox and Grid

Flexbox and CSS Grid are modern layout systems used to create responsive and well-structured web pages. They help you align, distribute, and organize elements more efficiently than older layout techniques.

Ways to add CSS

CSS can be added to a webpage in three ways:

  • Inline CSS – written directly in an HTML element using the style attribute.
  • Internal CSS – written inside a <style> element in the <head> section of the HTML file.
  • External CSS – written in a separate .css file and linked with <link rel="stylesheet">.

CSS properties

CSS properties control the appearance and layout of HTML elements. Common examples include color, background-color, margin, padding, border, display, justify-content, align-items, and grid-template-columns.

Flexbox

Flexbox is used for one-dimensional layouts. It is useful when you want to align items in a row or in a column, control spacing, and center content easily.

  • display: flex; – activates Flexbox
  • flex-direction – sets row or column layout
  • justify-content – aligns items on the main axis
  • align-items – aligns items on the cross axis
  • gap – creates space between items

Flexbox example

Example of a layout where an image and a text description are placed side by side:

.profile-section {
    display: flex;
    flex-direction: row;
    align-items: center;
    justify-content: space-between;
    gap: 20px;
}

CSS Grid

CSS Grid is used for two-dimensional layouts. It is useful when you want to organize content into rows and columns and build larger page structures.

  • display: grid; – activates Grid
  • grid-template-columns – defines the number and size of columns
  • grid-template-rows – defines the rows
  • gap – creates spacing between grid items

Grid example

Example of a layout with two equal columns:

.projects-container {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 20px;
}

When to use Flexbox and Grid

  • Use Flexbox for simple alignment, navigation bars, cards in a row, or centering content.
  • Use Grid for page sections, multi-column structures, galleries, or dashboard-like layouts.
  • Both can be combined in the same page when needed.

References: MDN – Flexbox, MDN – CSS Grid.

Task 1 – Accept Lab 5 assignment and clone Lab 4

Start by accepting the Lab 5 assignment on the course platform. While logged into GitHub, click Accept this assignment to create your Lab 5 private repository.

Next, open your local project for Lab 4 and make a clone in the Lab 5 repository. This cloned project will serve as the starting point for Lab 5. Ensure that the existing pages from Lab 4 are included, as you will modify them to add the required layout improvements for Lab 5.

Task 2 – Build layouts with Flexbox and Grid

For Lab 5, you will complete one exercise using Flexbox and CSS Grid. All layout rules must be written in an external stylesheet.

Exercise 1. – Create a CV page

Create a new page called cv.html and link it from your homepage.

Build your personal CV using CSS Grid for the main layout and Flexbox for smaller content areas.

  • The page must include: profile (name and short description), skills, projects, and contact details.
  • Use Grid to organise the main sections of the page.
  • The layout must not be a simple vertical list of elements.
  • Create a card-based section arranged in multiple columns, with at least 4 cards. Each card must contain a title and short text.
  • Use Flexbox inside the cards or inside another content area to align elements.
  • At least one card must be styled differently from the others.
  • Use spacing and alignment so the page looks clear, balanced, and suitable for a real CV / portfolio page.

Task 3 – Push, publish, and submit Lab 5

After completing the CV page and layout updates, make sure your repository for Lab 5 contains:

  • All files from your Lab 4 project.
  • cv.html – the new CV page created in this lab.
  • Any CSS files used for layout (for example: style.css).

Push your final changes to the Lab 5 repository. Open the live GitHub Pages URL and confirm that:

  • All pages load correctly.
  • The link to the CV page works.
  • The layout renders correctly, including alignment, spacing, and structure.

Task 4 – HTML validation

As the final step for Lab 5, validate your cv.html page using the W3C Markup Validation Service.

  • Fix all errors and warnings reported by the validator.
  • Check that the structure is correct and that the layout works properly.

Your Lab 5 deliverable is complete only when cv.html passes validation with no errors.
