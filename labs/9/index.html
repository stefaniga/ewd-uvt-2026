Lab 9 - Client-side form validation and interactive UI components

Duration: 2 hours. This lab introduces client-side form validation and interactive UI components using JavaScript.

In this lab, you will implement form validation on the contact page and add interactive UI components. You will learn to validate user input, provide feedback, and enhance user experience with dynamic elements.

On this page: Overview · Theory · Lab Setup · Tasks · Push, publish, and submit Lab 9 · Extra · References.

Overview

Objectives: Implement client-side form validation and create interactive UI components.

Concepts: Form validation, user input handling, interactive components, and user feedback.

Tasks: Validate the contact form and add interactive elements.

Deliverable: A validated contact form with interactive UI components.

Theory

Interaction: alert, prompt, confirm

Alert

Shows a message. This one we've seen already. It shows a message and waits for the user to press "OK".

Simple example:

alert("Hello");

Complex example:

let userName = "John";
let greeting = `Welcome, ${userName}!`;
alert(greeting);
let currentTime = new Date().toLocaleTimeString();
alert(`Current time: ${currentTime}`);
alert("Session started successfully!");

Prompt

Shows a message asking the user to input text. It returns the text or, if Cancel button or Esc is clicked, null. The function prompt accepts two arguments: result = prompt(title, [default]);

Simple example:

let age = prompt('How old are you?', 100);
alert(`You are ${age} years old!`);

Complex example:

let name = prompt("What is your name?", "Anonymous");
if (name) {
    let age = prompt(`Hello ${name}, how old are you?`, 25);
    if (age) {
        alert(`${name}, you are ${age} years old!`);
    } else {
        alert("Age not provided.");
    }
} else {
    alert("Name not provided.");
}

Confirm

Shows a message and waits for the user to press "OK" or "Cancel". It returns true for OK and false for Cancel/Esc.

Simple example:

let deleteItem = confirm("Are you sure you want to delete this item?");
alert(deleteItem ? "Item deleted" : "Deletion cancelled");

Complex example:

let proceed = confirm("Do you want to proceed with the action?");
if (proceed) {
    let action = prompt("What action to perform?", "Save");
    if (action) {
        alert(`Performing: ${action}`);
        // Simulate action
        alert("Action completed!");
    } else {
        alert("No action specified.");
    }
} else {
    alert("Action cancelled.");
}

Tasks

In this laboratory, you will start from the form created in the previous lab (Lab 2) in the contact.html file and add data validation using JavaScript. The goal is to prevent the form from being submitted if the entered data does not meet the defined rules.

To allow proper access and validation of the form fields, each input element must have a unique id attribute. These IDs will be used in the JavaScript file to select and validate the values entered by the user.

<input type="text" name="fullname" id="fullname">

The validation rules are as follows: the Full Name field must contain at least 5 characters and include only letters and spaces, without numbers or special characters. The Email must be valid and must end with the domain @e-uvt.ro. The Phone field is optional, but if filled in, it must contain exactly 10 digits. The Subject field is mandatory and must be selected from the available options, and the Message field cannot be empty. Additionally, the user must select at least one option from the radio button question.

Finally, if all validations are successfully passed, the user will receive a confirmation dialog before the form is submitted, asking for final approval. The form will only be submitted if the user explicitly confirms the action.

In addition to the existing requirements described above, you will extend the current form by adding new input fields in contact.html. These new fields must also follow the same principle: each element must have a unique id attribute in order to be accessible and validated through JavaScript.

<input type="date" id="dob">
<input type="number" id="age">
<input type="url" id="website">
<input type="file" id="fileUpload">
<input type="color" id="favColor">

The validation rules for these additional fields are as follows: the Date of Birth field must ensure that the user is at least 18 years old based on the selected date. The Age field must accept only values between 18 and 60. The Website URL must be a valid URL and must start with https://. The File Upload field must accept only .pdf or .docx files and the file size must not exceed 2MB. The Color input is mandatory and must not be left empty.

All validations must be implemented in JavaScript, and the form submission must be blocked if any of these conditions are not satisfied. Only when all fields (both existing and newly added) are valid, the form can proceed to the final confirmation step described in the previous task.
